Creating Gmail OAuth Credentials

Enable the app to scan your Gmail for PayPal notifications

Intermediate
1

Go to Google Cloud Console

Navigate to the Google Cloud Console and create a new project (or select an existing one).

Open Google Cloud Console
2

Create a New Project

Click on the project dropdown at the top of the page, then click "New Project".

  • Project Name: Choose something like "Raffle Email Scanner"
  • Organization: Leave as default (No organization)
  • Click "Create" and wait for the project to be created
3

Enable Gmail API

In your new project, you need to enable the Gmail API.

  • Go to APIs & Services ‚Üí Library
  • Search for "Gmail API"
  • Click on it and press "Enable"

üí° Tip: The Gmail API is free for personal use. You won't be charged for using it with this app.

4

Configure OAuth Consent Screen

Before creating credentials, you must configure the OAuth consent screen.

  • Go to APIs & Services ‚Üí OAuth consent screen
  • Select "External" user type (unless you have a Google Workspace)
  • Click "Create"
üéØ Required Fields
  • App name: "Raffle Email Scanner" (or your choice)
  • User support email: Your email address
  • Developer contact email: Your email address

Click "Save and Continue" through the remaining screens. You can leave most fields blank.

5

Add Gmail API Scope

On the "Scopes" step, you need to add the Gmail readonly scope.

  • Click "Add or Remove Scopes"
  • Search for "gmail"
  • Select: https://www.googleapis.com/auth/gmail.readonly
  • Click "Update" and then "Save and Continue"

‚ö†Ô∏è Important: Only select the "readonly" scope. This ensures the app can only read emails, not send or delete them.

6

Add Test Users (Optional)

If your app is in "Testing" mode, add your Gmail address as a test user.

  • On the "Test users" screen, click "Add Users"
  • Enter your Gmail address
  • Click "Save and Continue"
7

Create OAuth Client ID

Now you can create the actual OAuth credentials.

  • Go to APIs & Services ‚Üí Credentials
  • Click "Create Credentials" ‚Üí "OAuth client ID"
  • Application type: Select "Desktop app"
  • Name: "Raffle Desktop App" (or your choice)
  • Click "Create"
8

Download Credentials JSON

After creating the OAuth client, download the credentials file.

  • A popup will appear with your Client ID and Client Secret
  • Click "Download JSON"
  • Save the file somewhere safe - you'll need it for the next step

‚úÖ Success: You now have your OAuth credentials! The file will be named something like client_secret_xxxxx.json

9

Upload to Settings

Go to the Settings page in this app and upload your credentials file.

  • Navigate to Settings in the sidebar
  • Find the "Gmail OAuth Credentials" section
  • Click the upload box and select your downloaded JSON file
  • Wait for the success message
10

Authorize the App

The first time you use the "Scan Email" feature, you'll be asked to authorize the app.

  • Click "Scan Email" on the Active Raffle page
  • A browser window will open asking you to sign in to Google
  • Select your Gmail account
  • Click "Continue" when warned about an unverified app
  • Review the permissions and click "Allow"

‚ö†Ô∏è Warning: Google will show a warning that the app is "unverified". This is normal for personal projects. Click "Advanced" ‚Üí "Go to [App Name] (unsafe)" to continue.

üéâ Done! Your app is now authorized to scan Gmail. The token will be saved automatically.

Creating PayPal/Raffle Label in Gmail

Organize your PayPal raffle payment emails automatically

Beginner
1

Open Gmail

Log in to your Gmail account in your web browser.

üìß Open Gmail
2

Access Label Settings

Navigate to the labels management section.

  • Click the Settings gear icon in the top-right corner
  • Select "See all settings"
  • Go to the "Labels" tab
3

Create "PayPal/Raffle" Label

Create a single label called "PayPal/Raffle".

  • Scroll down to the "Labels" section
  • Click "Create new label"
  • Enter "PayPal/Raffle" as the name (include the forward slash)
  • Make sure "Nest label under" is unchecked
  • Click "Create"

‚úÖ Perfect! Your label PayPal/Raffle is now created and ready to use.

4

Create Filter to Auto-Label Emails

Set up a filter to automatically label PayPal payment notification emails.

  • Click the Settings gear icon in the top-right corner
  • Select "See all settings"
  • Go to the "Filters and Blocked Addresses" tab
  • Click "Create a new filter"
üéØ Filter Criteria
  • From: service@paypal.com
  • Subject: sent you
  • Click "Create filter"
  • Check the box for "Apply the label:"
  • Select "PayPal/Raffle" from the dropdown
  • Optional: Check "Also apply filter to matching conversations" to label existing emails
  • Click "Create filter"

üí° Pro Tip: Using "sent you" in the subject line specifically catches PayPal payment notifications like "John Doe sent you $50.00 USD"

5

Manually Label Existing Emails (Optional)

If you want to label PayPal emails you've already received:

  • Search for emails from PayPal: from:service@paypal.com
  • Select the emails related to raffle payments
  • Click the Labels icon (tag icon in toolbar)
  • Select "PayPal/Raffle"
  • Click "Apply"
6

Verify the Label

Check that the label appears correctly in Gmail.

  • Look in the left sidebar of Gmail
  • You should see "PayPal/Raffle" label
  • Click on it to view all labeled emails
7

Test with Email Scanning

Now test the email scanning feature in the app.

  • Go to the Active Raffle page
  • Click "Scan Email"
  • The app will scan emails in the PayPal/Raffle label
  • Matching payments will be automatically marked as paid

üéâ All Set! Your PayPal raffle emails are now organized and ready to be scanned automatically!

Troubleshooting

Common issues and solutions

‚ùå "API not enabled" error

Make sure you've enabled the Gmail API in Google Cloud Console (Step 3 above).

‚ùå "Access denied" when authorizing

If you selected "Internal" user type, make sure you're using a Google Workspace account. Otherwise, use "External" and add yourself as a test user.

‚ùå No emails found when scanning
  • Verify the PayPal/Raffle label exists and contains emails
  • Check that the label name is exactly PayPal/Raffle (case-sensitive)
  • Make sure at least one email has been labeled
‚ùå Payments not matching
  • Check that the participant names in your raffle match the PayPal sender names
  • Verify the payment amounts match exactly (including currency)
  • Use the Reddit Name Mappings feature in Settings to map Reddit usernames to real names

üìû Still need help? Contact support or check the documentation for more troubleshooting tips.
